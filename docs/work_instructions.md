# Инструкции по работе с проектом "АСУ ТП - Система управления устройствами"

## Введение

Проект "АСУ ТП - Система управления устройствами" представляет собой веб-приложение для управления списками устройств автоматизированных систем управления технологическими процессами (АСУ ТП). Приложение позволяет организовывать устройства в иерархическую структуру, редактировать их параметры и экспортировать данные в различные форматы (Excel, Word, PDF).

Основные возможности:
- Древовидное отображение устройств
- CRUD-операции с устройствами
- Поиск и фильтрация устройств
- Управление сигналами устройств
- Импорт/экспорт данных (Excel, PDF с поддержкой кириллицы)
- Много-проектная поддержка

Технологический стек:
- **Фронтенд**: React 19, TypeScript, Material-UI, Ant Design
- **Бэкенд**: Node.js, Express, TypeScript, Sequelize ORM
- **База данных**: SQLite
- **Дополнительно**: Axios, ExcelJS, jsPDF, Multer
- **Экспорт**: Excel (ExcelJS), PDF (Puppeteer)

## Предварительные требования

Для работы с проектом необходимо установить:

- **Node.js** версии 14 или выше (рекомендуется LTS)
- **npm** версии 6 или выше (поставляется с Node.js)
- **Git** для клонирования репозитория

Проверить установку можно командами:
```bash
node --version
npm --version
git --version
```

## Установка проекта

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd AsuOpt
```

### 2. Установка зависимостей

Установите зависимости для клиента и сервера:

```bash
# Зависимости клиента
cd client
npm install

# Зависимости сервера
cd ../server
npm install
```

### 3. Инициализация базы данных

База данных SQLite создается автоматически при первом запуске сервера. Для инициализации структуры выполните:

```bash
cd server
npm run build
npm run dev
```

При первом запуске будут созданы необходимые таблицы.

## Запуск и разработка

### Быстрый запуск

Используйте скрипт `restart_services.sh` для одновременного запуска клиента и сервера:

```bash
./restart_services.sh
```

Этот скрипт:
1. Исправляет базу данных (если необходимо)
2. Запускает сервер на порту 3001
3. Запускает клиент на порту 3000

### Ручной запуск

#### Запуск сервера (режим разработки)

```bash
cd server
npm run dev
```

Сервер запустится на http://localhost:3001

#### Запуск клиента (режим разработки)

```bash
cd client
npm start
```

Клиент запустится на http://localhost:3000

### Сборка для продакшена

#### Сборка сервера

```bash
cd server
npm run build
npm start
```

#### Сборка клиента

```bash
cd client
npm run build
```

Собранные файлы будут в папке `client/build`

### Режимы разработки

- **Сервер**: Использует nodemon для автоматической перезагрузки при изменениях в файлах `.ts`
- **Клиент**: React development server с hot reload
- **База данных**: SQLite файл `database.sqlite` в корне сервера

## Интерфейс пользователя

Приложение имеет следующую структуру вкладок:

1. **Устройства** - управление устройствами и их иерархической структурой
2. **Сигналы** - управление сигналами (сводная таблица и типы сигналов)
3. **Экспорт данных** - экспорт сигналов и устройств в Excel/PDF
4. **Импорт данных** - импорт данных из CSV файлов
5. **Управление БД** - операции с базой данных

## Работа с базой данных

### Модели данных

Основные таблицы:
- **devices**: Устройства с иерархией (parentId)
- **signals**: Справочник сигналов (AI, AO, DI, DO)
- **device_signals**: Связь устройств и сигналов
- **device_type_signals**: Количество сигналов по типам устройств
- **projects**: Проекты для разделения данных

### Миграции

Для применения миграций:

```bash
cd server
npm run migrate-projects        # Основные миграции
npm run migrate-projects-step   # Пошаговые миграции
```

### Импорт данных

```bash
npm run import-data             # Импорт временных файлов
npm run migrate-signals         # Миграция таблицы сигналов
```

### Очистка базы данных

```bash
npm run clear-db                 # Очистка всех таблиц
```

### Исправление базы данных

```bash
npm run fix-database             # Автоматическое исправление проблем
```

## Разработка новых функций

### Добавление нового устройства

1. Создайте интерфейс в `client/src/interfaces/Device.ts` (если необходимо)
2. Добавьте компонент формы в `client/src/components/`
3. Создайте API маршруты в `server/src/routes/deviceRoutes.ts`
4. Реализуйте контроллер в `server/src/controllers/deviceController.ts`
5. Добавьте модель в `server/src/models/Device.ts` (если необходимо)

### Добавление нового типа сигнала

1. Обновите справочник в таблице `signals`
2. Добавьте логику в `SignalManagement.tsx`
3. Обновите контроллер сигналов

### Добавление экспорта в новый формат

1. Установите библиотеку (например, для PDF - Puppeteer для лучшей поддержки кириллицы)
2. Создайте новый маршрут в `exportRoutes.ts`
3. Реализуйте логику в `exportController.ts` или отдельном модуле
4. Добавьте клиентскую поддержку в `exportService` и компоненты UI

**Примечание**: PDF экспорт использует Puppeteer для генерации HTML в PDF, что обеспечивает корректное отображение кириллических символов.

## Устранение неполадок

### Сервер не запускается

- Проверьте порт 3001 (может быть занят)
- Убедитесь, что база данных не повреждена: `npm run fix-database`
- Проверьте логи в `logs/` или консоль

### Клиент не подключается к серверу

- Проверьте, что сервер запущен на 3001 порту
- Проверьте CORS настройки в `server.ts`
- Убедитесь в правильности URL в `client/src/services/api.ts`

### Ошибки базы данных

- Очистите базу: `npm run clear-db`
- Перезапустите сервер для пересоздания таблиц
- Проверьте логи миграций

### Проблемы с импортом/экспортом

- Проверьте формат файлов (CSV, Excel)
- Убедитесь в наличии необходимых библиотек (exceljs, csv-parser)
- Проверьте права на запись в директорию uploads

### Общие советы

- Используйте инструменты разработчика браузера для отладки клиента
- Для API используйте Postman или curl
- Проверяйте логи сервера в консоли
- При проблемах с зависимостями удалите `node_modules` и переустановите
