# Руководство пользователя "АСУ ТП - Система управления устройствами"

## Введение

Добро пожаловать в систему управления устройствами АСУ ТП! Это веб-приложение предназначено для организации, управления и документирования устройств автоматизированных систем управления технологическими процессами.

### Основные возможности
- 📋 Организация устройств в иерархическую структуру
- 🔍 Мощные инструменты поиска и фильтрации
- 📊 Управление сигналами устройств
- 📄 Экспорт данных в Excel и PDF
- 👥 Поддержка многопроектной работы

---

## 1. Начало работы

### 1.1 Запуск системы

1. Убедитесь, что у вас установлены Node.js и npm
2. Клонируйте репозиторий проекта
3. Установите зависимости:
   ```bash
   # В директории client
   cd client && npm install

   # В директории server
   cd ../server && npm install
   ```
4. Запустите систему:
   ```bash
   # Из корневой директории
   ./restart_services.sh
   ```
5. Откройте браузер и перейдите на http://localhost:3000

### 1.2 Интерфейс приложения

Приложение состоит из следующих основных разделов:
- **Устройства** - управление устройствами и их структурой
- **Сигналы** - работа со справочником сигналов
- **Экспорт данных** - экспорт информации в различные форматы
- **Импорт данных** - загрузка данных из CSV файлов
- **Управление БД** - обслуживание базы данных

## 2. Работа с устройствами

### 2.1 Просмотр устройств

Главная страница отображает дерево устройств в иерархической структуре:

```
📁 Устройства
├── 🔧 PLC-001 (Программируемый логический контроллер)
│   ├── 📡 AI-001 (Аналоговый вход давления)
│   ├── 📡 DI-001 (Дискретный вход состояния клапана)
│   └── 📡 DO-001 (Дискретный выход управления насосом)
└── 🔧 PLC-002 (Резервный контроллер)
```

**Навигация:**
- Щелчок по папке разворачивает/сворачивает уровень
- Щелчок по устройству открывает подробную информацию

### 2.2 Поиск устройств

#### Глобальный поиск
- Расположен в верхней части страницы
- Поддерживает поиск по всем полям устройства:
  - Обозначение позиции
  - Описание
  - Тип устройства
  - Код системы
  - Тип ПЛК
  - Ex-версия
  - Данные КИП (производитель, единицы измерения и т.д.)
  - Данные ЗРА (тип клапана, диаметр и т.д.)

#### Примеры поиска:
- `PLC` - найдет все устройства с типом PLC
- `давление` - найдет устройства, связанные с измерением давления
- `Ex` - найдет взрывозащищенные устройства

### 2.3 Фильтрация устройств

Нажмите кнопку **"Фильтры"** для доступа к расширенным фильтрам:

#### Основные фильтры:
- **Тип данных**: КИП, ЗРА или неопределенный
- **Тип устройства**: PLC, Sensor, Valve и т.д.
- **Код системы**: фильтр по системам предприятия
- **Тип ПЛК**: Siemens, Schneider, Rockwell и т.д.
- **Ex-версия**: 0, 1, 2 (взрывозащита)

#### Специализированные фильтры КИП:
- Участок, площадка, производитель
- Единицы измерения, зона ответственности
- Схема подключения, питание, характеристики среды

#### Специализированные фильтры ЗРА:
- Тип конструкции, тип клапана, тип привода
- Номинальный диаметр, материал трубопровода
- Среда, датчик положения

#### Фильтры по датам:
- Дата создания (с/по)
- Дата изменения (с/по)

#### Работа с фильтрами:
1. Выберите нужные критерии
2. Нажмите **"Применить"**
3. Для сброса нажмите **"Сбросить"**
4. Фильтры можно сохранить как пресеты для быстрого доступа

### 2.4 Добавление устройств

1. Нажмите кнопку **"Добавить"** в панели управления
2. Заполните обязательные поля:
   - **Система**: код системы предприятия
   - **Оборудование**: код оборудования
   - **Линия**: номер технологической линии
   - **Шкаф**: название распределительного шкафа
   - **Обозначение**: уникальный идентификатор устройства
   - **Тип**: PLC, Sensor, Valve, Motor и т.д.
   - **Описание**: функциональное назначение
3. При необходимости заполните данные КИП или ЗРА
4. Нажмите **"Создать"**

### 2.5 Редактирование устройств

1. Выберите устройство в дереве
2. В панели деталей нажмите **"Редактировать"**
3. Измените необходимые поля
4. Сохраните изменения

## 3. Работа с сигналами

### 3.1 Справочник сигналов

Раздел "Сигналы" содержит две вкладки:

#### Типы сигналов
- Просмотр всех доступных типов сигналов
- Поиск по названию, типу, категории
- Добавление новых типов сигналов
- Импорт из CSV файлов

#### Сводная таблица
- Показывает количество сигналов по типам устройств
- Автоматическое заполнение для новых типов
- Возможность корректировки количества сигналов

### 3.2 Добавление типа сигнала

1. Перейдите на вкладку "Типы сигналов"
2. Нажмите **"Добавить сигнал"**
3. Заполните поля:
   - **Название**: уникальное имя сигнала
   - **Тип**: AI, AO, DI, DO
   - **Описание**: подробное описание
   - **Категория**: группа сигналов (Измерение, Управление, Защита)
   - **Тип подключения**: 2-проводное, 4-проводное
   - **Напряжение**: 4-20mA, 24V и т.д.

## 4. Импорт и экспорт данных

### 4.1 Импорт из CSV

1. Перейдите в раздел **"Импорт данных"**
2. Выберите тип импорта:
   - **Категории сигналов**
   - **Устройства**
   - **Назначение сигналов**
3. Выберите CSV файл
4. Нажмите **"Импортировать"**

### 4.2 Экспорт данных

1. Перейдите в раздел **"Экспорт данных"**
2. Выберите формат:
   - **Excel** - табличный формат с форматированием
   - **PDF** - печатная версия с кириллицей
3. Выберите данные для экспорта:
   - Устройства
   - Сигналы
   - Справочники
4. Нажмите **"Экспортировать"**

---

## 5. Управление проектами

### 5.1 Создание проекта

1. Перейдите в раздел управления проектами
2. Нажмите **"Создать проект"**
3. Заполните:
   - **Название**: понятное имя проекта
   - **Код**: уникальный код (например, ASU-2024-001)
   - **Описание**: подробная информация о проекте

### 5.2 Переключение между проектами

- Используйте селектор проекта в верхней части интерфейса
- Все устройства и сигналы привязаны к выбранному проекту
- Фильтры и поиск работают в рамках текущего проекта

## 6. Полезные советы

### 6.1 Горячие клавиши
- `Ctrl+F` - фокус на поисковом поле
- `Enter` - применение фильтров
- `Esc` - сброс фильтров

### 6.2 Оптимизация работы
- **Используйте фильтры** для быстрого нахождения нужных устройств
- **Сохраняйте часто используемые фильтры** как пресеты
- **Регулярно экспортируйте** важные данные для резервного копирования
- **Проверяйте актуальность** справочников сигналов

### 6.3 Поиск эффективных запросов
- **По типу**: `PLC`, `датчик`, `клапан`
- **По системе**: `Технологическая`, `Энергетическая`
- **По оборудованию**: `Насос`, `Теплообменник`
- **По параметрам**: `давление`, `температура`, `расход`

### 6.4 Устранение проблем
- **Данные не отображаются**: проверьте выбранный проект
- **Фильтры не работают**: нажмите "Сбросить" и попробуйте снова
- **Не удается сохранить**: проверьте заполнение обязательных полей
- **Поиск не находит**: попробуйте более короткие или общие термины

## 8. Поддержка

### 8.1 Часто задаваемые вопросы (FAQ)

**Q: Как добавить устройство в определенную систему?**
A: При добавлении устройства заполните поле "Код системы" соответствующим кодом предприятия.

**Q: Почему некоторые устройства не отображаются в поиске?**
A: Проверьте, что устройство относится к текущему выбранному проекту.

**Q: Как импортировать данные из существующей системы?**
A: Используйте CSV импорт в разделе "Импорт данных". Подготовьте файл согласно формату примера.

**Q: Можно ли работать с несколькими проектами одновременно?**
A: Да, но каждый пользователь работает в рамках одного выбранного проекта. Для одновременной работы над несколькими проектами используйте разные браузерные сессии.
